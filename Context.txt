Stage 4.2: AI Token Protection
This is important and should be early.

Build/verify:

Rate limit AI-heavy endpoints:
POST /api/interpret
POST /api/chat
prescription/entity extraction routes if exposed through upload
Use stricter limits for chat than upload/interpret.
Add max input length for chat messages.
Add max report history included in chat context, for example latest 10-20 reports or summarized context.
Add timeout and graceful fallback around Gemini calls.
Add user-facing fallback:
“AI is temporarily unavailable. Your structured report was still saved.”
Prevent repeated AI calls for the same uploaded report where possible.
Recommended implementation shape later:

express-rate-limit
route-specific limiters
bounded prompt builders
deterministic fallback summary for saved documents
optional hash-based AI result reuse later
Acceptance criteria:

A single user cannot spam Gemini through chat.
Demo does not fail hard if Gemini is slow/unavailable.
Structured extraction and persistence still work even if AI interpretation fails.

Day 5: Personal Health Intelligence Layer
Goal
Move from “dashboard” to “intelligence platform.”

Stage 5.1: Longitudinal AI Insights
This is the highest-value missing flagship feature.

Build an endpoint like:

GET /api/repository/insights or
POST /api/insights/longitudinal
Input should be structured, not raw OCR:

user profile
latest report
previous reports
trend series
active meds
diagnoses
abnormal markers
tests advised
doctor advice
Strict AI output schema:

{
  "summary": "",
  "trendInsights": [],
  "riskFlags": [],
  "doctorQuestions": [],
  "followUpSuggestions": [],
  "disclaimer": ""
}
Important language rules:

Say “may indicate,” “worth discussing,” “trend observed.”
Do not say “you have,” “start medication,” or “diagnosed with.”
Always include a doctor-facing disclaimer.
Acceptance criteria:

AI can say what changed over time.
It cites structured facts like dates, values, and markers.
It does not invent diagnoses.
It fails gracefully if Gemini quota is hit.
Stage 5.2: AI Insights UI
Create a premium dashboard section:

“What changed since your last report?”
“Needs attention”
“Improving”
“Questions to ask your doctor”
“Suggested follow-up”
Acceptance criteria:

It is visually prominent.
It is more impressive than a generic summary.
It makes the product feel personalized.
Stage 5.3: Doctor Summary Export
