Goal: move the product from “dashboard” to “health intelligence.”

Acceptance criteria:

- Know current test count.
- Know whether frontend build passes.
- Know current endpoint behavior.
- Do not duplicate existing functionality.

### Stage 1.2 — Longitudinal Insight Backend

Build an endpoint:

`GET /api/repository/insights`

or:

`POST /api/insights/longitudinal`

Preferred: `GET /api/repository/insights` because insights belong to the health repository.

Input data should come from structured records only:

- user profile
- latest report
- previous reports
- abnormal measurements
- trend series
- active/recent medications
- diagnoses
- symptoms
- doctor advice
- tests advised
- timeline events

Do not send raw OCR text to this endpoint.

Strict AI schema:

```json
{
  "summary": "",
  "whatChanged": [],
  "improvingSignals": [],
  "needsAttention": [],
  "riskFlags": [],
  "doctorQuestions": [],
  "followUpSuggestions": [],
  "disclaimer": ""
}
```

Language rules:

- Say “may indicate,” “trend observed,” “worth discussing.”
- Never say “you have disease X.”
- Never prescribe medication.
- Never tell the user to stop or start medicine.
- Always recommend doctor consultation for clinical decisions.

Fallback behavior:

If Gemini fails, return deterministic insights from existing structured data:

- abnormal markers
- latest value compared to previous value
- missing follow-up tests
- active medication count
- recent doctor advice

Acceptance criteria:

- Endpoint works with demo user.
- Endpoint does not hallucinate.
- Endpoint works without Gemini.
- Response is useful even with only 2 reports.
- Tests cover success and fallback.

### Stage 1.3 — Longitudinal Insight UI

Add a premium dashboard section:

Title options:

- “What Changed Since Your Last Report”
- “Health Intelligence Brief”
- “Longitudinal Health Insights”

Must show:

- one plain-language summary
- improving markers
- needs-attention markers
- questions to ask doctor
- suggested follow-up
- safety disclaimer

The UI should feel more important than a normal AI summary. This is the flagship feature.

Acceptance criteria:

- Evaluator instantly sees that HealthLens analyzes history, not just one report.
- Works with demo data.
- Has loading state.
- Has error/fallback state.
- Does not block the dashboard if insight API fails.

# Medical Safety Rules

Every AI-generated insight must follow these rules:

- Do not diagnose.
- Do not prescribe.
- Do not claim certainty.
- Do not replace clinical judgment.
- Use structured facts.
- Mention dates and values when possible.
- Recommend discussing concerns with a healthcare professional.

Required disclaimer:

> HealthLens AI provides informational insights based on uploaded records. It does not diagnose, prescribe treatment, or replace professional medical advice.
